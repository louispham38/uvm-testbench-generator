CLI Guide

Use UVM Testbench Generator locally via the command line — perfect for scripting, CI/CD pipelines, and offline workflows.

1 Installation

Option A: Install from source (recommended)

# Clone the repository git clone https://github.com/louispham38/uvm-testbench-generator.git cd uvm-testbench-generator # Install (creates the `uvmgen` command) pip install -e .

Option B: Install dependencies only

pip install -r requirements.txt python -m uvmgen.cli --help

Verify installation

uvmgen --version uvmgen, version 1.0.0

Requires Python 3.9+

2 Quick Start

Generate a complete AXI4-Lite UVM testbench in one command:

uvmgen generate --name axi_slave --protocol axi4_lite --output ./my_tb

This creates a ./my_tb/ directory with 16 files:

3 Commands

uvmgen generate

Generate a complete UVM testbench.

FlagDescriptionDefault
--name, -nProject / DUT namerequired
--protocol, -pProtocol typecustom
--data-width, -dwData bus width (bits)32
--addr-width, -awAddress bus width (bits)32
--clock-mhzClock frequency in MHz100
--tests, -tTest names (repeatable)base_test
--config, -cJSON config file (overrides all flags)—
--output, -oOutput directory./output

uvmgen init <filename>

Create a sample JSON config file to get started.

uvmgen init my_project.json Sample config written to my_project.json # Then customize and generate: uvmgen generate --name my_dut --config my_project.json --output ./tb

uvmgen list-protocols

Show all available built-in protocols.

uvmgen list-protocols Available Protocols: custom Custom protocol axi4_lite AXI4-Lite (AMBA) Protocol apb APB (AMBA Advanced Peripheral Bus) Protocol spi SPI (Serial Peripheral Interface) Protocol uart UART (Universal Asynchronous Receiver Transmitter) Protocol

uvmgen serve

Start the web UI server locally.

uvmgen serve --port 8000 Starting UVM Testbench Generator on http://0.0.0.0:8000 # With auto-reload for development: uvmgen serve --reload

--hostHost to bind0.0.0.0

4 Config File Format

Use a JSON config for full control. The web app's Save Config button exports the same format.

{ "project_name": "axi_slave", "module_name": "", "protocol": { "protocol": "axi4_lite", "data_width": 32, "addr_width": 32, "clock_freq_mhz": 100, "extra_params": {} }, "agent": { "is_active": true, "has_driver": true, "has_monitor": true, "has_sequencer": true, "has_coverage": true }, "tests": [ { "name": "sanity_test", "description": "Basic R/W", "num_transactions": 50, "timeout_ns": 5000 }, { "name": "stress_test", "description": "High throughput", "num_transactions": 500, "timeout_ns": 50000 } ], "components": { "interface": true, "sequence_item": true, "sequences": true, "driver": true, "monitor": true, "sequencer": true, "agent": true, "scoreboard": true, "coverage": true, "env": true, "test": true, "top": true, "package": true } }

Tip: Use uvmgen init config.json to generate a starter file, or Save Config from the web app.

5 Supported Protocols

axi4_liteAXI4-Lite

AMBA AXI4-Lite with full port set (AWADDR, WDATA, BRESP, ARADDR, RDATA, etc.), protocol constraints, and coverage.

apbAPB

AMBA APB with PSEL, PENABLE, PWRITE, PADDR, PWDATA, PRDATA, PREADY, PSLVERR.

spiSPI

Serial Peripheral Interface with SCLK, MOSI, MISO, CS_N. Configurable CPOL, CPHA, bit order.

uartUART

UART with TX, RX, CTS_N, RTS_N. Configurable baud rate, data bits, parity, stop bits.

customCustom

Define your own ports manually. No auto-generated ports or protocol-specific constraints — full flexibility.

6 Examples

Generate APB testbench with multiple tests

uvmgen generate \ --name apb_gpio \ --protocol apb \ --data-width 32 \ --tests sanity_test \ --tests stress_test \ --tests error_inject_test \ --output ./apb_gpio_tb

Generate from saved config (web app → CLI workflow)

# 1. Design in web app, click "Save Config" → my_dut_config.json # 2. Use that config locally: uvmgen generate --name my_dut --config my_dut_config.json --output ./tb

Generate SPI testbench with 8-bit data

uvmgen generate -n spi_flash -p spi -dw 8 -o ./spi_tb

Run the generated testbench

cd ./my_tb # The generated run.sh auto-detects VCS, Xcelium, or Questa chmod +x run.sh ./run.sh # Or manually with VCS: vcs -sverilog -ntb_opts uvm -f file.list +UVM_TESTNAME=base_test

7 CI/CD Integration

Automate testbench generation in your pipeline:

GitHub Actions example

# .github/workflows/gen-tb.yml name: Generate UVM TB on: [push] jobs: generate: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: actions/setup-python@v5 with: { python-version: '3.11' } - run: | pip install -e . uvmgen generate --name my_dut --config tb_config.json -o ./generated_tb - uses: actions/upload-artifact@v4 with: name: uvm-testbench path: ./generated_tb/

Makefile integration

# Makefile TB_DIR := ./tb CONFIG := tb_config.json gen-tb: uvmgen generate --name $(DUT) --config $(CONFIG) -o $(TB_DIR) sim: gen-tb cd $(TB_DIR) && ./run.sh
